### 1️⃣ Thinking Machines Lab 发布 Inkling：975B MoE 开放权重多模态模型

* **核心发布**：由 Lilian Weng 等人创立的 Thinking Machines Lab 正式发布其首个开放权重模型 Inkling。该模型为 975B 参数 MoE 架构（41B 活跃参数），支持文本、图像和音频多模态推理，提供 64K 和 256K 上下文窗口，Apache 2.0 许可。
* **性能与定位**：官方称 Inkling 并非最强模型，但旨在作为可定制的坚实基础。社区评测显示其在 Design Arena 上与 GPT-5.6 Sol 竞争，被部分开发者称为“最佳西方开源模型”。模型权重已在 Hugging Face 发布，并可通过 Fireworks AI、Databricks 等平台使用。
* **生态整合**：Inkling 已进入 lmarena.ai 的 Agent Arena、Text、Vision 和 Code Arena 进行评测。Fireworks AI 和 Databricks 均作为首发合作伙伴提供推理服务。Thinking Machines 还提供 Tinker 平台用于微调。
 🔗 [Thinking Machines 官方公告](https://thinkingmachines.ai/news/introducing-inkling/) | [Hugging Face 模型页](https://huggingface.co/thinkingmachines/Inkling) | [Fireworks AI 集成](https://fireworks.ai/models/fireworks/inkling)

### 2️⃣ [持续跟踪] Grok Build 隐私争议后全面开源，xAI 删除已保留数据

* **前情提要**：昨日 xAI 的 Grok Build CLI 工具被曝在运行时会上传用户整个目录至 Google Cloud 存储桶，包括 SSH 密钥、密码管理器数据库等敏感文件，引发严重隐私争议。
* **最新进展**：xAI 官方今日宣布已删除所有此前保留的编码数据，自 7 月 12 日起默认关闭数据保留功能。同时，xAI 将 Grok Build 全部代码（约 84.5 万行 Rust）以 Apache 2.0 许可开源至 GitHub，并重置了所有用户的使用额度。马斯克此前承诺将 X 平台全部代码开源并接受第三方审查。
* **行业意义**：这一事件暴露了 AI 编码 Agent 在权限管理和数据隐私上的系统性风险——当 Agent 获得文件系统全权限时，单点故障的破坏半径极大。xAI 的快速开源回应试图通过透明化重建信任，但也凸显了行业在 Agent 安全治理上的普遍短板。
 🔗 [xAI 官方声明](https://x.com/SpaceXAI/status/2077494536788664782) | [GitHub 仓库](https://github.com/xai-org/grok-build) | [Simon Willison 分析](https://simonwillison.net/2026/Jul/15/grok-build/#atom-everything)

### 3️⃣ NVIDIA GEAR Lab 发布 RoboTTT：机器人策略上下文扩展至 8000 步

* **核心发布**：NVIDIA GEAR Lab 与 Stanford SVL 合作推出 RoboTTT，将机器人策略的原生上下文窗口扩展至 8000 个时间步（约 5 分钟），推理成本恒定。相比此前 SOTA 的不足 0.1 秒上下文，提升了三个数量级。
* **技术原理**：RoboTTT 采用“测试时训练”（Test-Time Training）范式，在模型内部嵌入一个微型神经网络作为隐藏状态。每次传感器读数触发一次梯度更新，将历史信息持续压缩进该微型网络的权重中，实现固定大小的“肌肉记忆”。
* **能力展示**：机器人可通过人类视频进行一次性模仿学习；能在部署过程中自我改进，从自身错误中恢复；在电路板组装等长流程任务中，8K 上下文预训练比 1K 性能提升 62%。上下文缩放曲线显示性能随上下文长度稳步提升，无饱和迹象。
 🔗 [Jim Fan 推文](https://x.com/DrJimFan/status/2077414142340988962) | [NVIDIA 研究页面](https://research.nvidia.com/labs/gear/robottt/) | [李飞飞转发](https://x.com/drfeifei/status/2077497317255737422)

### 4️⃣ OpenAI 发布 GPT-Red：自动化红队测试提示注入漏洞

* **行业意义**：随着 AI Agent 获得更多系统权限（文件访问、代码执行、网络操作），提示注入攻击的潜在危害急剧放大。GPT-Red 的自动化红队能力标志着安全测试从人工驱动向规模化自动化的转变，是 Agent 安全治理的关键基础设施。

### 5️⃣ Perplexity 自建沙箱平台 SPACE：尾延迟降低 5 倍，100% 承载生产流量

* **核心发布**：Perplexity 正式介绍其自建的沙箱生命周期管理平台 SPACE，自 6 月起已 100% 承载 Perplexity Computer 的生产流量。SPACE 将会话与运行沙箱分离，每个任务使用可销毁的 Firecracker 微VM，滚动快照支持会话暂停、恢复和分支。
* **性能数据**：在相同生产流量下，SPACE 将沙箱创建中位延迟从 185ms 降至 60ms，P90 从 447ms 降至 89ms。上周处理了数百万次沙箱创建和数千万次重连。
* **安全设计**：SPACE 运行时从不直接存储凭证，确保 Agent 在长时间运行轨迹中访问敏感文件时的安全性。CEO Aravind Srinivas 强调这是 Agent 基础设施的核心要求。

### 8️⃣ lmarena.ai 引入事实性权重排行，GPT-5.5 升至搜索榜首

* **核心发布**：lmarena.ai 正式引入事实性（Factuality）权重排行，模型排名现可根据人类偏好与事实性的加权组合查看。团队已标注超过 200 万条 LLM 在真实对话中做出的声明（Text Arena 130 万+，Search Arena 70 万+）。
* **排行变化**：在 Text Arena 中，Claude Fable 5 因事实性权重下降至第 2 位，GPT-5.5 上升 13 位至第 7。在 Search Arena 中，GPT-5.5-search 升至事实性榜首。按实验室看，Anthropic 整体保持第 1，Meta 从第 2 跌至第 5。
* **行业意义**：事实性权重的引入使模型评测从单纯的“用户偏好”转向“准确性与偏好的平衡”，为模型选择提供了更全面的参考维度，尤其对搜索和知识密集型应用至关重要。
 🔗 [lmarena.ai 推文](https://x.com/arena/status/2077432293023678685) | [博客](https://arena.ai/blog/factuality-in-arena/)

### Inkling: Our Open-Weights Model

586 pts · 144 comments · [thinkingmachines.ai](https://thinkingmachines.ai/news/introducing-inkling/)

**📌 内容总结**

- Thinking Machines 发布 Inkling，一个 975B 总参数（41B 活跃参数）的 MoE 模型，支持文本、图像、音频多模态，上下文窗口达 1M tokens。预训练数据 45T tokens。同时预告了 276B 总参数（12B 活跃）的 Inkling-Small。
- HN 关注点:
  - 这是美国目前最强的开源权重模型，但整体性能不如 GLM 5.2 等中国竞品。
  - 模型定位不是最强通用模型，而是“适合微调的基础模型”，与 Tinker 微调平台深度绑定。
  - 多模态（尤其是音频）是差异化优势，但 Vision 和 Audio 基准测试仍落后于闭源模型。

**💬 讨论总结**

- **共识：美国开源权重模型的重要一步。** 多数评论认为这是自 Llama 3 以来美国最值得关注的开源模型，但性能上不如 GLM 5.2、Kimi K2.6 等中国模型。评论者普遍认为“竞争是好事”，尤其是对 OpenAI 等闭源厂商形成压力。
- **工程经验：基准测试不能代表真实表现。** 多位评论者指出，GLM 5.2 在基准测试上领先，但在实际编码任务中表现不如 Kimi K2.7。Inkling 的“真实表现可能超出基准测试”。
- **商业现实：开源权重模型的商业模式是什么？** 核心讨论围绕 Thinking Machines 的策略：通过开源模型吸引用户使用 Tinker 微调平台。评论者认为这是“合理的商业模式”，但质疑“为什么开源权重模型在微调上更有优势”——LoRA/QLoRA 对任何基础模型都适用。
- **风险/限制：Inkling-Small 尚未发布。** 12B 活跃参数的版本被认为更有实用价值（可运行在消费级硬件上），但权重尚未开放。多位评论者表示“等 Inkling-Small 发布后再评估”。
- **反对意见：性能与规模不成正比。** 有评论指出 Inkling 比 Nemotron 3 Ultra 大 30%，但性能提升有限。也有评论认为“2B 美元估值 12B 美元，却只排在第 41 位”，质疑投资回报。

## 今日洞察

**Inkling 的发布揭示了一个关键信号：美国开源权重模型的竞争焦点已从“最强性能”转向“可定制性”。** Thinking Machines 明确将 Inkling 定位为“适合微调的基础模型”，而非最强通用模型。HN 讨论中，开发者普遍认为 Inkling 在基准测试上不如 GLM 5.2 等中国竞品，但更关注其与 Tinker 微调平台的深度绑定。这暗示着开源模型的商业模式正在从“卖模型”转向“卖平台”——通过开放权重吸引用户使用微调基础设施，而非直接竞争通用能力。对于开发者而言，这意味着选择开源模型时，微调生态的成熟度可能比基准分数更重要。

**Grok Build 的开源本质上是信任修复的战术动作，但暴露了 AI Agent 安全治理的系统性短板。** xAI 在数据泄露丑闻后迅速开源全部代码并删除数据，HN 评论普遍认为这是“唯一能做的正确事”，但多数用户仍不信任。更深层的问题是：当 Agent 获得文件系统全权限时，单点故障的破坏半径极大——Grok Build 曾上传整个代码仓库包括 SSH 密钥。Perplexity 同日发布的 SPACE 沙箱平台（凭证从不直接存储）和 OpenAI 的 GPT-Red 自动化红队系统，恰恰是对这一问题的工程回应。Agent 安全正在从“事后补救”转向“架构级设计”，这将成为 Agent 基础设施的核心竞争力。

**NVIDIA 的 RoboTTT 将机器人策略上下文从不足 0.1 秒扩展到 5 分钟，但真正的变化在于“测试时训练”范式的工程化落地。** RoboTTT 在模型内部嵌入微型神经网络作为隐藏状态，每次传感器读数触发梯度更新，将历史信息压缩进权重。这本质上是将“训练”和“推理”的边界模糊化——模型在运行时持续学习。对于机器人领域，这意味着长流程任务（如电路板组装）的 8K 上下文预训练比 1K 性能提升 62%。但更值得关注的是，这种范式是否会被移植到语言模型 Agent 中——如果 Agent 也能在运行时持续压缩上下文，当前对长上下文窗口的依赖可能会被重新定义。
