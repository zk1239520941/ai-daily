Perplexity 开源 WANDR：内部研究基准，评估 Agent 的广度与深度搜索能力

  • 核心发布：Perplexity 正式开源 WANDR，这是其内部用于构建 Perplexity Computer 深度和广度研究能力的基准。WANDR 包含 500 个研究任务，需 170,495 个源支持记录，分三级难度。
  • 评测机制：不同于对标准答案评分，WANDR 重新获取每个引用页面并检查每项声明与底层证据的一致性，支持时间变化的事实。软评分给部分分数，硬评分要求每个成员的所有组件完整且正确。
  • 行业意义：WANDR 不仅是一个基准，其管道还可作为半自动化的训练数据工厂。Perplexity CEO 称这是其内部评估中表现最佳的基准，为 Agent 研究能力提供了可复现的评测标准。

🔟 Google 与行业伙伴发布 Agentic Resource Discovery 开放标准

  • 核心发布：Google 与行业合作伙伴宣布 Agentic Resource Discovery (ARD) 规范，这是一个用于发布、发现和验证 AI 工具、API 和 Agent 的开放标准。ARD 引入了基于目录和注册表的发现层，支持动态能力发现，同时利用现有协议（如 MCP 和 OpenAPI）进行执行。

Bonsai 27B: A 27B-Class model that runs on a phone

369 pts · 140 comments · prismml.com

📌 内容总结

  • PrismML 发布基于 Qwen3.6 27B 的极端量化模型，提供三元（1.71 bits/weight, 5.9GB）和 1-bit（1.125 bits/weight, 3.9GB）两个变体，后者首次让 27B 参数模型在 iPhone 17 Pro 上运行。
  • 在 15 项基准测试中，三元变体保留 95% 的完整精度性能，1-bit 变体保留 90%。数学和编码能力损失较小，工具调用和视觉任务下降更明显。
  • 核心技术是端到端低比特表示（嵌入、注意力、MLP、LM head 均使用低比特），配合 FP16 分组缩放，无高精度逃逸通道。支持 262K token 上下文和推测解码。
💬 讨论总结
  • 共识观点： 极端量化在数学和编码任务上表现尚可，但工具调用和 agentic 任务的质量下降在实际使用中可能更显著。多位用户提到 BFCL v3 多轮结果能更好反映真实场景中的“死循环”问题。
  • 工程经验： 手机实际可用内存远低于标称值（12GB iPhone 约 6GB 可用），模型还需与 KV cache 和激活值共享预算。1-bit 变体约 4GB 是首次能通过这个门槛的 27B 模型。
  • 历史背景： 多位用户指出“1-bit”命名有历史误导性——最早的 1-bit 模型实际使用 {+1,0,-1} 三值。Bonsai 同时提供真正的二元（{+1,-1}）和三值变体。
  • 反对/质疑：
  • 有用户报告在 Android 上仅输出“!!!!!!!!!!!!!”，认为这是 Qwen 系列过度量化的典型失败模式。
  • 独立评测者（lm-evaluation-harness + vllm）发现 Bonsai 的 4-bit 变体在 wikitext 困惑度上从 8.0 飙升至 16.75，GSM8K 得分为 0，怀疑评测工具兼容性问题或量化过度。
  • 部分用户认为 27B 参数对手机应用而言过大，压缩后的性能可能不如专门微调的小模型。
  • 商业现实： PrismML 获 Khosla Ventures、Cerberus、Google 和 Samsung 支持。CNBC 报道 Apple 正与 PrismML 洽谈，但 CEO 公开透露此事可能影响合作关系。

Cursor 0day: When Full Disclosure Becomes the Only Protection Left

214 pts · 89 comments · mindgard.ai

📌 内容总结

  • Mindgard 发现 Cursor IDE（Windows 版）在加载项目时会搜索多个路径寻找 git.exe，其中包括当前工作目录。攻击者可在仓库根目录放置恶意 git.exe，Cursor 会自动执行，无需用户交互。
  • 该漏洞于 2025 年 12 月 15 日报告，经过 7 个月、197+ 个版本后仍未修复。Cursor 的 CISO 曾回应称自动化流程故障导致未及时处理，但后续沟通中断。
  • 漏洞利用简单：克隆恶意仓库 → 用 Cursor 打开 → 自动执行。演示中 Cursor 会反复执行该二进制文件（非一次性触发）。
💬 讨论总结
  • 共识观点： 漏洞本身简单且严重，但更令人担忧的是 Cursor 长达 7 个月的不作为。多位用户认为这是 AI 辅助开发工具安全流程失效的典型案例。
  • 工程经验： Windows 默认将当前目录加入 PATH，这是许多类似漏洞的根源。Unix 默认不在 PATH 中包含“.”正是为了防范此类问题。
  • 历史背景： 多位用户将此事与 Windows AutoPlay 漏洞类比——微软曾因 USB 自动执行恶意软件而禁用该功能。Cursor 的行为本质上类似。
  • 反对/质疑：
  • 部分用户认为攻击者需先让受害者下载恶意文件到本地，这本身已有一定门槛。
  • 有用户指出 VSCode 的 SSH 和 devcontainer 远程功能同样允许 RCE，此类问题并非 Cursor 独有。
  • 多位用户批评博客文章本身疑似 AI 生成，冗长且信息密度低，可能影响可信度。
  • 商业现实： Cursor 估值约 600 亿美元，年收入 40 亿美元，拥有 700 万+活跃用户。有用户猜测公司忙于 SpaceX 收购等事务而忽视安全。

Show HN: Juggler – an open-source GUI coding agent, by the creator of JUCE

163 pts · 77 comments · github.com/juggler-ai

  • 作者想做什么： 由 JUCE 和 Tracktion 创建者 Jules 开发的 GUI 编码代理，提供可视化工作台，让用户更精细地控制 LLM 对代码库的操作。

  • 产品设计： 支持本地/远程/多客户端同时连接，可接入 Claude、OpenAI、Gemini、Ollama、DeepSeek 等多种模型。所有工具（读/写/bash）均为可替换的插件。
  • 用户反馈焦点： UI 设计获得普遍好评，树状会话结构被认为是比“末日滚动”更好的交互范式。
💬 讨论总结
  • 共识观点： 树状会话结构是编码代理 UI 的正确方向，多位用户表示这是他们最想要但现有工具缺乏的功能。AGPL 许可证和 40MB 的小体积也获得正面评价。
  • 工程经验： 作者透露该项目约 100 万行代码，但几乎全部由 AI 生成。他发现自己享受的是最终结果而非编码过程本身。
  • 历史背景： 多位用户提到 JUCE 是音频插件开发的事实标准，作者在实时系统领域的背景让社区对项目架构质量有信心。
  • 反对/质疑：
  • 部分用户希望支持 ACP（Agent Communication Protocol）以复用现有代理生态。
  • 有用户报告 DeepSeek 4 Pro 连接时出现 reasoning_content 格式错误。
  • 作者承认沙箱支持、技能扫描等功能尚在开发中。
  • 商业现实： 该项目为个人业余时间开发，无 VC 支持，无商业计划。作者表示若用户基数足够大，会考虑全职投入。
