Apple's new SpeechAnalyzer API, benchmarked against Whisper and its predecessor

412 pts · 172 comments · get-inscribe.com

📌 内容总结

  • 作者运营一款同时集成 Apple 新旧语音引擎和 WhisperKit 的转录应用，借此在相同生产代码路径下，对五个引擎进行了 LibriSpeech 基准测试（5,559 条语料，全设备端运行于 M2 Pro）。
  • Apple 未公布 SpeechAnalyzer 的精度数据，迁移决策此前依赖猜测。测试结果：新 API 在干净语音上 WER 2.12%（旧 API 9.02%），噪声语音 4.56%（旧 API 16.25%），同时以约 3 倍于 Whisper Small 的速度运行并击败后者。
  • 作者公开了所有原始逐句转录结果，Whisper 列与 OpenAI 公布数据偏差在 +0.11 到 +0.42 之间，验证了测试方法的可靠性。局限性：仅限英语、朗读语音、单机测试。
💬 讨论总结
  • 共识观点：Apple 新 API 在英语语音识别上确实显著优于旧 API，且对大多数用户而言，迁移是明确的。
  • 工程经验：
  • 旧 API 默认将音频发送至 Apple 服务器，需手动设置 requiresOnDeviceRecognition 才能强制设备端运行。
  • 新 API 使用后必须调用 finalizeAndFinishThroughEndOfInput() 才能结束会话，否则会永久挂起——作者在自己的产品中发现了这个 bug。
  • 历史背景：Whisper Small/Base/Tiny 已发布近四年，未随 Whisper v2/v3 更新，并非当前最优模型。多位评论者指出应对比 Parakeet、Nemotron、Voxtral、Cohere Transcribe 等更近期的模型。
  • 反对意见：
  • 仅对比 Whisper 小模型（Small/Base/Tiny）而不包括 Large V3 或 V3 Turbo，测试不完整。Whisper Large V3 Turbo 可在旧款 MacBook 上实时运行且精度更高。
  • 多语言支持是 Whisper 的核心优势（100+ 语言 vs. SpeechAnalyzer 约 30 个区域），且 Whisper 可跨平台运行。
  • 有评论者认为 Parakeet TDT 0.6b v2/v3 在精度和速度上已优于 Whisper，且可在旧款 iPhone 15 Pro Max 上良好运行。
  • 商业现实：对开发者而言，使用 Apple 内置模型的最大优势是不需将模型打包进应用，节省磁盘空间；用户只需下载一次，所有应用共享。

Show HN: Nobie – an Excel-compatible runtime for agents and humans

  • 解决的问题：Mac 上缺乏优秀的 Excel 替代品；现有方案要么不兼容，要么需要上传数据到云端。
  • 技术实现：支持所有 Excel 公式、样式、表格、图表、条件格式、LET/LAMBDA/动态数组等；渲染引擎可生成像素级精确的 PNG 和 PDF；支持增量重算。
  • 产品设计：本地优先，数据不离开设备；提供 CLI、MCP 接口，可直接与 Claude/Codex/Gemini 等 AI 代理交互；免费使用。
  • 用户反馈焦点：社区关注其与 Excel 的兼容性、性能、以及对 CSV 导入、欧洲数字格式等细节的支持。
💬 讨论总结
  • 共识观点：社区对一款现代、本地优先、兼容 Excel 的 Mac 原生应用有明确需求。
  • 工程经验：
  • 引擎完全自研，不依赖 Apache POI 等库，目标是比 Excel 更快。
  • 作者将 Nobie 与 Excel 的关系类比为 V8 与 JavaScript：Nobie 是运行时，而 openpyxl 只是文件格式库，无法维护公式依赖、图表等语义。
  • 商业现实：免费模式引发对商业可持续性的疑问。作者未明确说明盈利方式。
  • 风险/限制：目前缺少透视表编辑、VBA/宏、多人协作等功能；Windows 版本尚未发布。
  • 反对意见：有评论者质疑“本地优先”与“支持 Claude/Codex”之间的矛盾——使用这些 AI 服务时数据仍会发送到第三方。
