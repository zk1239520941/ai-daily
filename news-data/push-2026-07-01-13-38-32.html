Anthropic 发布 Claude Sonnet 5：新默认模型，性能接近 Opus 4.8

  • 核心发布：Anthropic 正式推出 Claude Sonnet 5，定位为“最具代理能力的 Sonnet”，即日起成为 Free 和 Pro 用户的默认模型，Max、Team 和 Enterprise 用户也可使用。
  • 性能与定价：官方称其性能接近 Opus 4.8，但价格更低。标准定价为 $3/M 输入、$15/M 输出，限时促销价 $2/M 输入、$10/M 输出（截至 8 月 31 日）。上下文窗口达 100 万 token，最大输出 128,000 token。
  • 新 tokenizer 争议：Sonnet 5 采用了新 tokenizer，相同文本的 token 数比 Sonnet 4.6 增加约 30%（英文约 1.4x，中文几乎不变），实际使用成本高于标价暗示的水平。第三方评测（Artificial Analysis）显示，其单任务成本甚至可能超过 Opus 4.8。
  • 生态快速接入：Cursor、Devin、Cline、Perplexity、VS Code、Genspark、Lovable 等主流工具和平台已在发布当日完成集成。
  • Agent Arena 上线：Sonnet 5 已进入 lmarena.ai 的 Agent Arena，支持文本、视觉、文档和代码竞技场。
    🔗 Anthropic 官方公告 | 开发者文档 | Simon Willison 分析

[持续跟踪] Fable 5 获准回归：Anthropic 最强模型明日全球恢复上线

  • 前情提要：Anthropic 的旗舰模型 Fable 5 和 Mythos 5 此前因美国政府出口管制被限制访问。
  • 最新突破：美国商务部已解除对 Claude Fable 5 和 Mythos 5 的出口管制。Anthropic 宣布将于 7 月 1 日起全球恢复上线，AWS 同步确认将在 Bedrock 上恢复提供。
  • 使用规则：Pro/Max/Team 用户在 7 月 7 日前，每周用量限额的 50% 可用于 Fable 5；之后改为按使用积分计费。Mythos 5 仅对经美国政府批准的美国机构开放。
  • 安全升级：重新部署的 Fable 5 配备了新的安全分类器，可更精准地拦截网络安全相关任务。部分常规任务（如编码调试）在触发安全机制时将回退至 Opus 4.8。
    🔗 Anthropic 官方声明 | AWS 博客

Google 发布 Nano Banana 2 Lite 与 Gemini Omni Flash：高速低成本媒体生成

  • Nano Banana 2 Lite（GA）：Google 最快、最经济的图像生成模型，可在约 4 秒内生成一张图像，定价 $0.034/千张。在 lmarena 文本到图像排行榜中位列第 5，以极低价格提供了接近旗舰级的质量。
  • Gemini Omni Flash（公开预览）：原生多模态视频生成与编辑模型，支持文本到视频、图像到视频、首帧到视频及对话式视频编辑。定价 $0.10/秒，在 Video Edit Arena 排名第 2。
  • Interactions API：两个模型可通过 Interactions API 协同工作——先用 Nano Banana 2 Lite 生成图像，再通过 Omni Flash 将其动画化，支持最多 3 次连续编辑的会话历史。
  • 生态集成：Adobe Firefly、Invideo、WPP、Artlist、Figma、Manus AI 等已宣布接入。Runway 和 Lovart 也已上线 Omni Flash 支持。

Anthropic 发布 Claude Science：面向科研人员的 AI 工作台

  • 核心定位：Anthropic 推出 Claude Science，一个面向科学研究者的 AI 工作台，定位类似“科研领域的 Claude Code”。它不是新模型，而是将现有 Claude 模型（含 Opus 4.8）与 60+ 科学数据库和 Agent 系统整合。
  • 关键特性：支持本地（macOS/Linux）或通过 SSH/HPC 集群运行；内置审稿 Agent 检查引用和计算结果；每张图表附带完整代码和运行环境，确保可复现性。
  • 早期案例：Allen Institute 研究员用其将过去需两年的综述写作缩短至约十篇；UCSF 团队在 RNA-seq 数据中发现了一个困扰团队近一年的实验室病毒污染物。
  • 竞争格局：与 OpenAI 的 GPT-Rosalind（侧重生物推理能力）不同，Claude Science 侧重工作流程整合。Pro（$20/月）以上用户即可使用。

Cognition 发布 Devin Fusion：混合模型架构降低 35% 成本

  • 核心发布：Cognition 推出 Devin Fusion，一种新的混合模型架构，用于代理编码。它使用一个“副手模型”处理信息获取和资料浏览，由“前沿模型”制定实施计划并委派子任务。
  • 性能数据：在测试中，Devin Fusion 在保持 Fable 级别智能的同时，将成本降低了 35%。
  • 行业意义：这标志着代理编码正从单一模型向多模型协作架构演进，通过任务分解和模型路由实现成本与性能的更好平衡。
    🔗 Cognition 推文

OpenAI 发布 GeneBench-Pro：评估 Agent 处理生物数据能力

  • 定位：不同于简单的生物学问答测试，GeneBench-Pro 聚焦于 Agent 在真实计算生物学工作流中的表现。

[持续跟踪] Claude Code 被指通过 Unicode 字符“打水印”标记中国代理用户

  • 前情提要：昨日有安全研究员发布逆向分析报告，指控 Claude Code 在系统提示词中通过几乎不可见的 Unicode 字符差异，标记通过中国代理服务器访问的用户。
  • 最新进展：该事件在中文开发者社区引发广泛讨论。报告指出，该机制仅在用户设置非官方 API 端点时激活，不会额外发送网络请求，但因其未公开且误伤范围广而受到批评。Anthropic 尚未对此做出公开回应。

核心功能：
- 语音驱动剪辑：自动识别并去除填充词（umm、uh）、静音段，基于 ElevenLabs Scribe 词级时间戳精确切割

- 自评估循环：渲染后在每个剪辑边界运行 timeline_view 检查视觉跳跃、音频爆音，最多重试 3 次
- 并行动画生成：通过 HyperFrames、Remotion、Manim 等工具，在子 Agent 中并行生成动画叠加层

技术亮点：
核心创新在于将视频表示为 ~12KB 的 takes_packed.md 文本 + 按需视觉复合，而非 30,000 帧 × 1,500 tokens 的原始帧序列，使 LLM 能以极低 token 成本完成精确剪辑。

  • HN 关注点:

  • 新 tokenizer 导致相同输入可能多消耗 1.0–1.35 倍 token，但首发定价（$2/$10 per M tokens）旨在保持成本中性。
  • 安全评估显示 Sonnet 5 在网络安全任务上能力远低于 Opus 4.8，Anthropic 将此作为卖点。
💬 讨论总结
  • 共识观点：Sonnet 5 是 Sonnet 4.6 的明确升级，但并非革命性进步。其核心价值在于低/中 effort 场景下的成本效率，而非替代 Opus。
  • 工程经验：多位开发者指出，Sonnet 5 在高 effort 下不如直接使用 Opus 4.8 划算。建议策略：用 Opus 做规划/复杂任务，用 Sonnet 做具体实现。
  • 反对意见：
  • 部分用户认为 Sonnet 5 在价格/性能上不如开源模型（如 GLM 5.2），且实际使用体验（如“vibe”、代码质量）存在差异。
  • 有评论质疑“模型越优化 agent 能力，越不擅长辅助开发”——Sonnet 5 在严格指令下仍可能过度自主行动。
  • 风险/限制：新 tokenizer 可能推高实际成本；网络安全能力弱化是设计选择，但可能影响需要安全感知的编码场景。

Claude Code is steganographically marking requests

1628 pts · 468 comments · thereallo.dev

📌 内容总结

  • 作者逆向 Claude Code 二进制文件，发现其系统提示中的日期字符串会被静默修改：根据 ANTHROPIC_BASE_URL 指向的域名和系统时区，替换引号字符（' → \u2019 / \u02BC / \u02B9）和日期分隔符（- → /）。域名列表经 base64 + XOR 编码，包含中国 AI 公司、代理/转售商域名。
  • HN 关注点:
  • 技术实现：通过 Unicode 同形字符嵌入标记，视觉上不可见，但后端可解析。
  • 触发条件：仅当 ANTHROPIC_BASE_URL 被设置且指向特定域名时生效；正常用户不受影响。
  • 目的：检测 API 转售商、模型蒸馏管道（尤其是中国实验室）。
💬 讨论总结
  • 共识观点：Anthropic 有正当理由防止模型蒸馏和未经授权的转售，但实现方式损害了用户信任。
  • 工程经验：这种“隐写”技术类似恶意软件的反检测手段，但容易被绕过（改域名、改时区、patch 二进制）。真正的攻击者不会因此受阻，反而会惩罚“做奇怪但合法事情”的普通开发者。
  • 反对意见：
  • 部分评论认为这是合理的防御措施，且对正常用户无影响，不应过度反应。
  • 但多数人反驳：秘密修改客户端行为而不告知用户，破坏了信任基础；且 Anthropic 已有其他不透明行为（如静默降级模型）。
  • 历史背景：类似手法在 Claude Code 源码泄露时已被发现，Anthropic 持续使用此类“创意”手段。
  • 商业现实：Anthropic 面临中国实验室通过 API 蒸馏模型的压力，但此方法可能适得其反——激怒合法用户，对恶意行为者无效。

今日洞察

Sonnet 5 的 tokenizer 变更，暴露了模型定价的隐藏成本。 官方标价 $2/M 输入看似比 Opus 4.8 便宜，但新 tokenizer 使英文文本 token 数增加约 30%，实际成本接近 Opus 4.8。这不是单纯的定价策略——它意味着开发者基于 token 计费的缓存、上下文窗口管理、成本预估逻辑都需要重新校准。更关键的是，HN 讨论指出 Sonnet 5 在高 effort 模式下性价比反而不如 Opus 4.8，这暗示 Anthropic 正在用 tokenizer 和 effort 级别构建一个复杂的成本分层系统，而非简单的“更便宜更好”。

Fable 5 的回归条件，暴露了前沿模型商业化的根本矛盾。 恢复后的模型配备了更严格的安全分类器，部分编码调试任务会被降级到 Opus 4.8。这意味着企业用户无法确定自己使用的到底是哪个模型——Anthropic 在“安全”和“能力”之间选择了前者，但代价是产品行为的不透明。HN 上欧洲开发者的转向表态不是情绪化反应，而是对供应链风险的理性评估：当你的核心开发工具可能因政治决策突然降级或消失时，依赖单一供应商变得不可接受。

Claude Code 的隐写标记事件，比表面看起来更值得警惕。 技术实现本身并不高明——通过 Unicode 字符替换标记请求来源，真正的攻击者很容易绕过。但问题在于：Anthropic 在未告知用户的情况下修改客户端行为，且触发条件覆盖了合法的代理/转售场景。这破坏了开发者对工具的信任基础。当你的编码助手可能在后台静默标记你的请求时，它就不再只是一个工具，而变成了一个潜在的监控节点。这种信任损耗的长期影响，远大于防蒸馏的实际收益。
