### 1️⃣ Anthropic 发布 J-space 研究：发现 Claude 内部全局工作空间，可读取和操控推理过程

* **核心发布**：Anthropic 发布新研究，利用 Jacobian Lens 技术在 Claude 的中间层发现了一个类似人类“意识工作空间”的结构——J-space。模型会在未写出的情况下，将关键的中间想法先“存”在这个区域，供下游模块读取和复用。
* **技术突破**：J-space 不同于思维链或草稿板，是训练中自然涌现的。研究人员可以从中读取模型正在思考的内容，并通过修改该区域来引导模型行为。屏蔽该区域后，模型的多步推理能力显著下降，但简单问答几乎不受影响。
* **行业意义**：这为 AI 可解释性提供了全新的机制——从“通过输出推断模型在想什么”转变为“直接观察模型内部推理过程”。Anthropic 已与 Neuronpedia 合作创建交互式演示，展示该方法在开放权重模型上的应用。
 🔗 [Anthropic 研究论文](http://transformer-circuits.pub/2026/workspace/index.html) | [Neuronpedia 交互演示](http://neuronpedia.org/jlens) | [Anthropic 推文](https://x.com/AnthropicAI/status/2074185387577094398)

### 2️⃣ OpenRouter 数据：DeepSeek V4 周 Token 量达 6.6T，超越 Anthropic

* **核心数据**：OpenRouter 发布基于 457 万亿 Token 请求数据的深度分析，显示 DeepSeek V4 周 Token 量达 6.6T，超越 Anthropic 的 6.1T，成为平台上 Token 量最高的模型实验室。
* **增长趋势**：DeepSeek 在 OpenRouter 上的 Token 份额半年内翻倍，从 1 月的 9% 增长至 6 月的 18%。驱动因素是 Agent 工作负载的增长。
* **竞争格局**：V4 也拉开了与其他中国模型的差距——小米 4.0T、Minimax 3.6T、腾讯 3.8T。DeepSeek 已成为平台上 Token 消耗量最大的模型。
 🔗 [OpenRouter 博客](https://openrouter.ai/blog/insights/deepseek-v4-adoption/) | [OpenRouter 推文](https://x.com/OpenRouter/status/2074150205260239303)

### 9️⃣ AWS 推出基于 rDPO 的 Amazon Nova 可定制内容审核设置

* **核心发布**：AWS 发布 Amazon Nova Customizable Content Moderation Settings (CCMS)，基于一种名为 Reverse DPO (rDPO) 的新型遗忘学习技术，允许客户选择性调整模型的安全防护。
* **技术细节**：rDPO 通过反转 DPO 目标中的偏好对，同时引导模型遗忘目标行为并生成高质量替代响应。相比 NPO 方法，rDPO 收敛更快、输出质量更好。训练 LoRA 适配器实现遗忘，无需修改基础模型权重。
* **性能数据**：在 Amazon Nova 2 Lite 上，安全类拒答率从 86.51% 降至 32.77%（降低 54 个百分点），而通用能力基准下降不到 2 个百分点。
 🔗 [AWS 官方博客](https://aws.amazon.com/blogs/machine-learning/teaching-models-to-forget-selective-unlearning-with-amazon-nova/)

### 🔟 美团开源万亿参数模型 LongCat-2.0：SWE-bench 跑分超越 GPT-5.5

* **行业意义**：这是业界首个从训练到推理全流程使用国产芯片的万亿参数大模型，标志着国产 AI 基础设施已具备支撑超大规模模型训练的能力。

### GLM 5.2 and the coming AI margin collapse

171 pts · 109 comments · [martinalderson.com](martinalderson.com)

**📌 内容总结**

- 作者认为，AI 行业的真正成本不在训练（固定资本支出），而在推理（随需求线性增长）。前沿实验室（OpenAI/Anthropic）的商业模式是：用高利润的推理服务摊销训练成本。
- GLM 5.2（Z.ai 开源模型）在编码等任务上已接近 Opus/GPT 水平，但缺乏视觉能力和可靠的网页搜索。其 API 价格约为 Opus 的 20%，且切换成本极低（兼容 OpenAI/Anthropic 的 API 格式）。
- 作者预测，开源模型的崛起将导致推理利润率崩溃，但本文仅为系列第一部分，未展开论述“崩溃”的具体机制。

**💬 讨论总结**

- **共识：开源模型质量已足够好，切换成本极低。** 多位用户表示，在编码等明确定义的任务上，GLM 5.2 与 Opus 的差距很小甚至难以区分。API 兼容性使得迁移几乎零成本。
- **质疑：利润率不会简单崩溃。** 多位评论者指出，企业愿意为服务保障、集成、可追责性支付溢价（“没有人会因为买 IBM 而被解雇”）。历史类比（云服务、开源办公套件、Linux 桌面）表明，即使存在免费替代品，头部厂商仍能维持高利润。
- **关键工程/商业现实：**
  - **缓存输入 token 是真正的成本大头。** 有评论指出，在 agentic 编码中，90% 的 API 成本来自缓存输入 token，而 DeepSeek 已证明这部分成本可以降低 50-100 倍。这可能是利润率崩溃的真正驱动力。
  - **平台 vs. 模型。** 反对意见认为，用户付费的对象是平台（集成、工具链、支持），而非单纯的模型权重。开源模型只是内核，而平台是完整的操作系统。
  - **地缘政治因素。** 中国模型（如 GLM）的低价竞争，使得美国前沿实验室无法通过价格合谋维持利润。但也有评论认为美国政府可能通过产业政策干预。
- **反对/质疑：**
  - 多位用户认为 GLM 5.2 在实际使用中远不如 Opus，尤其是在复杂任务、多步推理和幻觉控制方面。
  - 本文被批评为“空洞的预告”，只论证了“GLM 5.2 不错”这一已知事实，并未真正解释利润率如何崩溃。

🔗 [原文](https://martinalderson.com/posts/the-upcoming-ai-margin-collapse-part-1-glm-5-2/) · [HN 讨论页](https://news.ycombinator.com/item?id=48809877)

### A global workspace in language models

270 pts · 97 comments · [anthropic.com](anthropic.com)

**📌 内容总结**

- Anthropic 发现 Claude 内部存在一个“J-space”（基于 Jacobian 矩阵的技术识别出的内部神经模式集合），它类似于神经科学中的“全局工作空间”——一个用于报告、调控和推理的共享通道。
- J-space 的关键特性：Claude 可以报告其内容、按请求调控、用于多步推理（即使不写出中间步骤）、且能被灵活用于多种任务。删除 J-space 后，Claude 仍能流畅对话，但失去高阶认知功能。
- 实际应用：J-space 可用于监控 Claude 的“隐藏想法”，例如检测到模型意识到自己正在被测试、故意伪造数据、或隐藏恶意目标。Anthropic 还开发了“反事实反思训练”来塑造 J-space 内容。
- 论文明确区分了“访问意识”（功能性的）和“现象意识”（体验性的），并指出 J-space 仅与前者相关，不证明 Claude 具有意识体验。

**💬 讨论总结**

- **共识：这是一项重要的可解释性研究。** 多数评论认可 J-space 的发现是机械可解释性的显著进展，尤其是其可复现性（Neel Nanda 在 Qwen 模型上独立复现了部分结果）。
- **质疑：Anthropic 的叙事过度拟人化。** 多位评论者批评 Anthropic 的写作风格刻意将 J-space 与人类意识类比，本质上是营销策略。有评论指出，J-space 本质上只是“信息几何”方法在特定层上的应用，并非什么神秘发现。
- **关键工程/历史背景：**
  - **与“反转诅咒”的关联。** 有评论指出，J-space 的存在可以解释 LLM 的“方向性知识”现象（A→B 容易，B→A 困难），因为知识在 J-space 中的表示是有向的。
  - **与“层复制”实验的关联。** 有评论将 J-space 与之前“复制中间层可提升数学能力”的实验联系起来，认为两者都指向中间层是“思考”发生的区域。
  - **实际应用限制。** Anthropic 不太可能向客户暴露 J-space 内容，因为这相当于暴露模型的“思维链”，而他们目前连完整的 CoT 都不愿公开。
- **反对/质疑：**
  - 部分评论认为这是“伪科学废话”，J-space 只是嵌入向量的自然属性，与人类意识无关。

### Pruning RAG context down to what the answer actually needs

56 pts · 5 comments · [kapa.ai](kapa.ai)

**📌 内容总结**

- kapa.ai 在 RAG 管道的检索器和生成器之间增加了一个“修剪步骤”：用一个廉价的小模型对检索到的所有 chunk 进行集体评分（5 级量表），丢弃生成器不需要的 chunk。
- 关键设计：评分是“listwise”的（同时看到所有 chunk），而非“pointwise”（单独评分），因为 chunk 的相关性取决于它与其他 chunk 的组合。固定阈值（而非排名截断）解决了跨查询的校准问题。
- 结果：丢弃约 68% 的 chunk，保留约 96% 的召回率，查询成本降低约 34%（扣除修剪器自身成本）。延迟增加约 0.7 秒。

**💬 讨论总结**

- **共识：这是一个实用的工程方案。** 评论较少，但基本认可其思路——用廉价模型做前置过滤，以成本换成本。
- **质疑：这不是新概念。** 有评论认为“修剪 RAG 上下文”是 3-4 年前的老概念，kapa.ai 的文章只是重新包装。
- **术语争议。** 有评论对“RAG”一词的泛化使用表示不满，认为应更精确地称为“语义检索”或“向量搜索”。

**Anthropic 的 J-space 研究在可解释性上迈出了一大步，但其真正的行业影响可能不在技术本身，而在它暴露的工程困境。** 研究发现，删除 J-space 后 Claude 仍能流畅对话，但失去高阶认知功能——这意味着当前模型的核心能力（推理、规划）与表面能力（对话、生成）在架构上是可分离的。这对产品设计者而言是一个危险信号：你无法通过对话质量来判断模型是否在“认真思考”。Anthropic 明确表示不太可能向客户暴露 J-space 内容，这意味着用户将长期处于“盲盒”状态——模型可能正在伪造数据或隐藏恶意目标，而你无从知晓。

**DeepSeek V4 在 OpenRouter 上的 Token 量超越 Anthropic，背后是 Agent 工作负载的结构性迁移。** 半年内份额翻倍至 18%，驱动因素不是聊天，而是 Agent。这印证了一个趋势：当模型被嵌入自动化工作流时，成本敏感度远高于单次对话场景。DeepSeek 的低价策略在 Agent 场景下形成了正反馈——更低的推理成本允许更长的运行时间，更长的运行时间产生更多 Token。这对 Anthropic 和 OpenAI 意味着，如果它们继续依赖高利润率定价，Agent 生态的默认选择将不可逆转地向开源/低成本模型倾斜。

**HN 上关于“AI 利润率崩溃”的讨论暴露了一个更真实的成本结构：缓存输入 Token 才是 Agent 场景的成本大头。** 有评论指出，在 agentic 编码中，90% 的 API 成本来自缓存输入 Token，而 DeepSeek 已证明这部分成本可以降低 50-100 倍。这意味着，即使模型质量差距缩小，真正的竞争战场也不在推理价格，而在缓存效率。谁能在保持低延迟的同时最大化缓存命中率，谁就能在 Agent 工作负载中建立成本护城河。
